# Requirements: SocraticDev Planning Scope

**Defined:** 2026-03-27
**Core Value:** Help developers learn through guided thinking, interactive practice, and context-aware feedback.

## v1 Requirements

### Landing Dojo Preview

- [x] **LDP-01**: Visitors can switch between exactly five landing-page dojo previews using clickable selector cards.
- [x] **LDP-02**: Each landing-page preview lets the visitor complete one compact real interaction with mode-specific validation.
- [x] **LDP-03**: After submission, the landing-page preview shows an immediate correct/incorrect verdict plus a short explanation.
- [x] **LDP-04**: After finishing a preview, visitors can either retry, continue to another preview, or enter the full `/dojo` route.
- [x] **LDP-05**: The landing preview stays aligned with existing landing-page responsiveness, accessibility, and reduced-motion behavior.

## v2 Requirements

### Deferred Landing Ideas

- **LDP-V2-01**: Show all ten dojo modes on the landing page.
- **LDP-V2-02**: Make timers, streaks, or scoreboard elements the main landing feedback surface.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Replacing the full `/dojo` experience | The landing page is an on-ramp, not a full dojo shell |
| Adding new dojo challenge types | Phase scope is limited to existing five chosen modes |
| Building persistent preview progress | Not required for the landing teaser experience |
